# LazyVim 中文使用指引

这份指引按 vimplus 用户迁移来写，目标是先把日常编辑跑顺，再慢慢理解 LazyVim。

## 安装思路

当前仓库是 `LazyVim/LazyVim` 的 fork，本体不应该直接作为 `~/.config/nvim` 使用。真正可用的个人配置在 `user-config/` 目录。

Windows 推荐安装位置：

```powershell
$env:LOCALAPPDATA\nvim
```

复制配置：

```powershell
Rename-Item $env:LOCALAPPDATA\nvim nvim.bak -ErrorAction SilentlyContinue
Copy-Item -Recurse .\user-config\* $env:LOCALAPPDATA\nvim\
nvim
```

## 常用入口

| 操作 | 快捷键 / 命令 |
| --- | --- |
| 查看可用快捷键 | 按 `<leader>` 后等待 which-key |
| 插件管理 | `:Lazy` |
| LazyVim extras | `:LazyExtras` |
| 工具安装 | `:Mason` |
| 健康检查 | `:checkhealth` 或 `:LazyHealth` |
| 格式化当前文件 | `<leader>cf` |
| 保存 | `<C-s>` |

`<leader>` 默认是空格。

## 目录和文件

| 操作 | 快捷键 |
| --- | --- |
| 打开/关闭项目目录树 | `<leader>e` 或 `<leader>nt` |
| 打开/关闭当前目录树 | `<leader>E` 或 `<leader>nT` |
| 查找项目文件 | `<leader>ff` |
| 查找当前目录文件 | `<leader>fF` |
| 最近文件 | `<leader>fr` |
| 搜索全文 | `<leader>sg` |
| 搜索当前光标单词 | `<leader>sw` |
| 新建文件 | `<leader>fn` |

目录树里常用操作：`l` 打开，`h` 折叠，`Y` 复制路径，`O` 用系统程序打开。

## IDE 布局

这套配置现在更接近 VS Code：

- 左边：项目目录树（`neo-tree`）
- 底部：问题列表 / 调用关系 / quickfix（`Trouble`）
- 右边：终端 / Codex CLI（`Snacks terminal`）

常用面板键位：

| 操作 | 快捷键 |
| --- | --- |
| 打开/关闭项目目录树 | `<leader>e` |
| 右侧终端（项目根目录） | `<leader>tt` |
| 右侧终端（当前目录） | `<leader>tT` |
| 右侧 Codex CLI（项目根目录） | `<leader>ac` |
| 右侧 Codex CLI（当前目录） | `<leader>aC` |
| 底部问题列表 | `<leader>cp` |
| 底部 quickfix | `<leader>cq` |
| 底部 Incoming calls | `<leader>ci` |
| 底部 Outgoing calls | `<leader>co` |

右侧终端里按两次 `<Esc>` 可以退出 terminal insert 模式，回到普通模式。

## 分屏、标签和 buffer

| 操作 | 快捷键 |
| --- | --- |
| 垂直分屏 | `<leader>sv` 或 `<leader>|` |
| 水平分屏 | `<leader>sh` 或 `<leader>-` |
| 关闭当前分屏 | `<leader>sc` 或 `<leader>wd` |

| 关闭 tab | `<leader><tab>d` |

## 跳转和代码阅读

| 操作 | 快捷键 |
| --- | --- |
| 跳到定义 | `gd` 或 `<leader>cd` |
| 查找引用 | `gr` 或 `<leader>cr` |
| 跳到实现 | `gI` |
| 跳到类型定义 | `gy` |
| 当前文件符号 | `<leader>ss` |
| 工作区符号 | `<leader>sS` |
| 代码大纲 | `<leader>cs` |
| Incoming calls（底部面板） | `<leader>ci` |
| Outgoing calls（底部面板） | `<leader>co` |
| Incoming calls（列表选择器） | `gai` |
| Outgoing calls（列表选择器） | `gao` |

调用层级依赖语言服务器支持。C/C++ 的 clangd 通常最好，Go 的 gopls 也比较可用，Python 取决于 pyright 能拿到多少静态信息。

## 对齐

视觉模式选中多行后按 `ga`，再输入分隔符。

例子：

```text
foo=1
long_name=2
```

选中两行后输入 `ga=`，会按 `=` 对齐。

## Go / Python / C++

配置已导入：

| 语言 | LazyVim extra | 主要工具 |
| --- | --- | --- |
| Go | `lazyvim.plugins.extras.lang.go` | `gopls`、`goimports`、`gofumpt`、`delve` |
| Python | `lazyvim.plugins.extras.lang.python` | `pyright`、`ruff`、`debugpy` |
| C/C++ | `lazyvim.plugins.extras.lang.clangd` | `clangd`、`clang-format`、`codelldb` |

第一次打开项目后执行：

```vim
:Mason
:Lazy sync
:checkhealth
```

C++ 项目最好提供 `compile_commands.json`，否则 clangd 对大型项目的跳转和补全会打折扣。
